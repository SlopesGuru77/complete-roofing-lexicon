### Root cause

`zbpack.json` ‚Üí `"output_dir": "public"` is **only applied when there is a `build_command`**. For pure-static no-build plans (this repo), zbpack's generated Dockerfile is:

```dockerfile
FROM alpine AS output
COPY . /                                  # <-- entire repo
FROM docker.io/zeabur/caddy-static:latest
COPY --from=output / /usr/share/caddy
```

Confirmed in the build log: `#6 [output 1/1] COPY . /`. `output_dir` is silently ignored for this code path. (Source: zbpack docs say `output_dir` is "where built files go" ‚Äî implicitly assumes there's a build step that produces them.)

## 5. The lesson worth keeping

Three distinct Zeabur quirks surfaced this session:

1. **`zbpack.json` ‚Üí `output_dir` is a no-op for pure-static no-build deploys.** Confirmed in attempt 1 of the first session. The auto-generated Dockerfile is two stages (`output` doing `COPY . /` and `runtime` doing `COPY --from=output / /usr/share/caddy`) ‚Äî `output_dir` only changes anything if there's a `build_command` producing files into it.
2. **Zeabur's "pre-processing Dockerfile" step rewrites Dockerfiles that `FROM zeabur/caddy-static`** ‚Äî substituting its own two-stage template and discarding explicit COPY allowlists. The pre-processor uses base-image identity as a signal that this is a managed static deploy. Avoid `FROM zeabur/caddy-static` if you want a non-template Dockerfile.
3. **Vanilla `FROM nginx:alpine` (or any non-Zeabur base) is left alone.** This is the escape hatch. The build log changes from "pre-processing Dockerfile ‚Üí 2.93 kB substitute" to a normal Docker build of exactly the lines I wrote.

**Hard rule for adding new static assets:** edit `Dockerfile`. The `COPY` line is an explicit allowlist ‚Äî anything not named there will not ship to the production host. That's the desired ergonomic now: surface decisions are explicit, not accidental.

**Hard rule against build steps still applies** in the spirit it was written ‚Äî no transpilation, no bundling, no per-source-file transforms. A two-line `COPY` directive isn't a build; nothing is being compiled or processed. The app is still buildless.
